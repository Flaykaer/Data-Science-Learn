1. Разобраться с основополагающими сущностями обмена данными между клиентом и сервером: протокол HTTP, заголовки и тело запроса, заголовки и тело ответа, статус-коды, IP-адрес и порт, формат данных JSON.

2. По ссылке (http://codepen.io/yaplusplus/pen/PWdOWN) приведен код страницы, которая взаимодействует с API Яндекс.Переводчика. 

Еще раз о том, что такое API (https://mkdev.me/posts/chto-takoe-api-v-veb-prilozheniyah-i-zachem-on-nuzhen).

Нужно модифицировать страницу таким образом, чтобы она принимала от пользователя три параметра: слово для перевода (поле для ввода); исходный язык данного слова (выпадающий список) и язык, на который нужно перевести слово (выпадающий список). После ввода данных и нажатия кнопки “Перевести”, страница должна отправлять запрос к API Яндекс.Переводчика, получать ответ и выводить его на экран (или текст ошибки в случае ее возникновения). 

Важно: чтобы работать с API Яндекс.Переводчика, нужно получить ключ доступа. 

Порядок действий:

получить и сохранить ключ доступа (https://tech.yandex.ru/keys/get/?service=trnsl) (нужно иметь аккаунт на Яндексе);

изучить приведенный пример программы и заменить в JS-коде значение переменной API_KEY на свой ключ;

написать свой вариант, который будет принимать слово для перевода от пользователя.

При необходимости можно смотреть в документацию по API (https://tech.yandex.ru/).
